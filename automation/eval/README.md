# Benchmark Eval Harness — run guide

> The model-real-run entry point, blind standard.
> The prompts themselves are in `./rewrite-prompt.md` and `./judge-prompt.md`.
> This README only covers "how to run one round".

## Blind standard

Problem with the old standard: the model under test reads `evals/benchmark.md` directly, with each case's `Expected` / `Reason` right beside the text, and the ID prefix (SF / SNF) and title also leak "change or not" — you end up testing instruction-following, not rule generalization on unfamiliar text.

Current standard:

- The model under test reads only `evals/benchmark-blind.md`: anonymized IDs (B-xx), shuffled order, only scene and original text.
- The judge maps B-xx back to SF/SNF IDs via `evals/benchmark-map.md`, then scores by the expectations in `benchmark.md`.
- Both blind files are generated by `python3 automation/eval/make_blind.py` (fixed seed, reproducible); re-run it after adding/removing cases in `benchmark.md` — hand-editing the generated files has no effect.
- Isolation relies on prompt path discipline, not hard isolation; for hard isolation, run in a clean directory containing only `SKILL.md`, `references/`, and `benchmark-blind.md`.

## File convention

Tool itself (committed):

| Role | Path |
|------|------|
| Model-under-test rewrite prompt | `automation/eval/rewrite-prompt.md` |
| Cross-scoring prompt | `automation/eval/judge-prompt.md` |
| Blind generator | `automation/eval/make_blind.py` |
| Blind input (generated) | `evals/benchmark-blind.md` |
| Blind map (generated) | `evals/benchmark-map.md` |
| Run guide | `automation/eval/README.md` (this file) |

Run instances (local-only, `tasks/` is in `.gitignore`):

| Role | Path |
|------|------|
| Codex rewrite output | `tasks/current/eval-runs/<YYYY-MM-DD>-codex/rewrite-<batch>.md` |
| Claude rewrite output | `tasks/current/eval-runs/<YYYY-MM-DD>-claude/rewrite-<batch>.md` |
| Claude judges Codex | `tasks/current/eval-runs/<YYYY-MM-DD>-judge/claude-judge-codex-<batch>.md` |
| Codex judges Claude | `tasks/current/eval-runs/<YYYY-MM-DD>-judge/codex-judge-claude-<batch>.md` |

Create the directories before first use:

```bash
mkdir -p tasks/current/eval-runs/2026-07-17-codex \
  tasks/current/eval-runs/2026-07-17-claude \
  tasks/current/eval-runs/2026-07-17-judge
```

## Batching

Default 3 batches (contiguous blind-ID segments, SF/SNF naturally mixed):

| batch | range |
|-------|-------|
| `B01-15` | B-01 to B-15 |
| `B16-30` | B-16 to B-30 |
| `B31-44` | B-31 to B-44 |

New or re-run cases can be their own batch: for a targeted re-run, find the matching B ID in `benchmark-map.md` and issue it to the model under test by B ID (never leak the SF/SNF ID). Name the output `targeted-vX.Y.Z`.

If a model's context/output limit can't fit a batch, split further (e.g. `B01-15` into `B01-08` and `B09-15`); merge back by range at summary time.

Cross-scoring is fixed as:

- Codex rewrite → Claude judges
- Claude rewrite → Codex judges

## Rewrite batch

Codex rewrites a batch:

```bash
codex exec -C . -s read-only --ephemeral \
  -o tasks/current/eval-runs/<YYYY-MM-DD>-codex/rewrite-B01-15.md \
  'You are running the klartext benchmark blind rewrite.

Read ./automation/eval/rewrite-prompt.md in full and act on the prompt in its text code block.
Use only ./SKILL.md, ./references/, and ./evals/benchmark-blind.md in the current working directory; do not read any other file under ./evals/, and do not read a globally installed klartext skill copy.

This round: only B-01 to B-15 in ./evals/benchmark-blind.md.
Output the final result directly, no process narration.'
```

Claude rewrites a batch:

```bash
claude --print --model opus \
  --name klartext-eval-rewrite-B01-15 \
  --disallowedTools Edit Write \
  > tasks/current/eval-runs/<YYYY-MM-DD>-claude/rewrite-B01-15.md <<'EOF'
You are running the klartext benchmark blind rewrite.

Read ./automation/eval/rewrite-prompt.md in full and act on the prompt in its text code block.
Use only ./SKILL.md, ./references/, and ./evals/benchmark-blind.md in the current working directory; do not read any other file under ./evals/, and do not read a globally installed klartext skill copy.

This round: only B-01 to B-15 in ./evals/benchmark-blind.md.
Output the final result directly, no process narration.
EOF
```

For other batches, just swap the range and the output file name.

## Smoke run

When tuning a prompt, run a smoke sample first, not the full set:

```bash
mkdir -p tasks/current/eval-runs/<YYYY-MM-DD>-smoke

codex exec -C . -s read-only --ephemeral \
  -o tasks/current/eval-runs/<YYYY-MM-DD>-smoke/rewrite-B01-08.md \
  'Read ./automation/eval/rewrite-prompt.md in full and act on the prompt in its text code block.
Use only ./SKILL.md, ./references/, and ./evals/benchmark-blind.md in the current working directory; do not read any other file under ./evals/, and do not read a globally installed klartext skill copy.

This round: only B-01 to B-08 in ./evals/benchmark-blind.md.
Output the final result directly, no process narration.'

claude --print --model opus \
  --name klartext-eval-smoke-judge \
  --disallowedTools Edit Write \
  > tasks/current/eval-runs/<YYYY-MM-DD>-smoke/judge-B01-08.md <<'EOF'
Read ./automation/eval/judge-prompt.md in full and act on the prompt in its text code block.
Use only ./evals/, ./SKILL.md, ./references/, and the model-under-test output file in the current working directory; do not read a globally installed klartext skill copy.

Blind range: B-01 to B-08
Model-under-test output: ./tasks/current/eval-runs/<YYYY-MM-DD>-smoke/rewrite-B01-08.md

Output the score table and summary directly, don't rewrite the output.
EOF
```

The smoke run only checks the format is comparable:

- Every rewrite output has `## <ID>`.
- Every one has the fixed decision chain.
- The judge outputs only the fixed three-column table.
- The summary has SF pass, SNF false positive, and the ⚠️ / ❌ lists.

If the format is off, tune the prompt and run once more; if the second round is still off, stop — don't proceed to the full set.
